git fork - serve para fazer alterações no projeto sem afetar o repositório original.
git clone - serve para clonar um reposítorio existente já existente.
git add - adicionar, alterar ou excluir arquivos locais para a staging area.
git commit - indica as alterações que você fez no seu projeto. Realização de um conjunto de mudanças, alterações que você realizou em seu projeto e ajuda no controle de versão. Para ser realizado, precisa de um git add anteriormente.
git push - O comando git push é usado para gravar em um repositório remoto, para acontecer é necessario ter realizado um commit e um add anteriormente.
